Part 5

Compliance Requirements

CMS audits DST-SEP documentation heavily. Every enrollment must pass this 5-step verification. No exceptions.

⚠️ Audit Risk

If CRM documentation doesn't show the full verification below, CMS auditors flag it immediately. Non-compliant enrollments result in plan disenrollment and carrier penalties.

  • 1

    Verify Residency

    Proof (ID, utility bill, lease) OR verbal attestation accepted per CMS/Humana/Wellcare. Document which was provided.

  • 2

    Confirm Missed Enrollment Period

    Agent must ask AND document which enrollment period was missed and why. Required — not optional.

  • 3

    Document in CRM (Word-for-Word)

    Exact format: "DST-SEP [STATE]. Beneficiary attests residency in [COUNTY] during [INCIDENT DATE RANGE]. [PROOF TYPE]. Confirmed missed enrollment: [WHICH]. Election code: SEP DST. Agent: [NAME]. Date: [MM/DD/YYYY]."

  • 4

    One Election Per SEP

    Enroll, disenroll, or switch — only ONE action per SEP event.

  • 5

    Correct Election Code + Never Cold-Market

    Election code: SEP DST (Humana/Wellcare). Per Humana: "Never use as a marketing tool. Wait for inbound interest."

Inbound Call Compliance

  • Consumer-initiated only — beneficiary must initiate call from a CMS-compliant lander
  • SMID required on all vendor creatives
  • TPMO disclaimer + explicit Medicare Advantage language on all materials
  • Call recordings retained by vendor
  • Jornaya LeadID required for lead-gen flows (where applicable)
  • Minimum 90-second call duration for billable
  • Hours of operation: Monday–Friday 8am–5pm MST
